# Project Charter: T&A Supply Gap Map

- 문서 버전: 0.1.0
- 상태: 재기획 초안
- 작성일: 2026-07-18
- 대상 기업/직무 맥락: 마이리얼트립 T&A실 해외 사업 개발 매니저
- 제품 형태: 사업개발 분석 MVP
- 출발점: 기존 `여행 경험 진열대` Codex 플러그인의 BD 분석도구 전환
- 한 줄 정의: 마이리얼트립 MCP 상품 데이터와 지자체·관광청 공식 관광 자산을 경험 단위로 대조해, 도시별 T&A 상품 공급이 충분한 영역과 공식 매력 대비 상품화가 부족한 영역을 찾는 사업개발 분석 도구

## 1. 프로젝트 배경

기존 `여행 경험 진열대` 플러그인은 여행자가 목적지를 입력하면 마이리얼트립 상품에 흩어진 장소·음식·체험을 `뭐 하지`, `뭐 먹지` 단위로 펼쳐주는 도구였다. 이 아이디어를 웹 서비스로 확장하는 과정에서 스탬프, 여행 스토리, 타임라인, 숙소 비교까지 범위가 넓어졌지만, T&A 해외 사업개발 매니저 직무 관점에서는 초점이 흐려질 수 있다.

이 프로젝트는 방향을 다시 좁힌다. 여행자용 앱을 만드는 것이 아니라, T&A 사업개발자가 도시별 상품 공급과 공식 관광 자산 사이의 차이를 빠르게 파악하는 분석 MVP를 만든다. 핵심 질문은 “이 도시에서 이미 마이리얼트립 상품으로 잘 팔 수 있는 경험은 무엇이고, 공식 관광 자산으로 확인되지만 아직 상품화가 부족한 경험은 무엇인가?”이다.

## 2. 문제 정의

### 2.1 사업개발 문제

T&A 사업개발자는 신규 지역, 신규 카테고리, 현지 파트너, 프로모션 기회를 찾아야 한다. 하지만 실제 업무에서는 다음 문제가 생긴다.

- 도시별 마이리얼트립 상품은 `투어`, `입장권`, `패스`, `미식`, `액티비티` 같은 판매 형식으로 흩어져 있다.
- 지자체·관광청은 명소, 축제, 음식, 체험, 로컬 브랜드를 공식적으로 소개하지만, 이 자산이 플랫폼 상품과 얼마나 연결되는지 한눈에 보기 어렵다.
- 공급이 많은 대도시와 공급이 적은 소도시는 같은 방식으로 분석하면 안 된다.
- 공식 관광 자산은 강하지만 플랫폼 상품이 적은 지역을 발견해야 신규 파트너 소싱과 상품 기획으로 이어질 수 있다.

### 2.2 기존 플러그인에서 배운 점

기존 플러그인은 “상품명 뒤에 숨은 경험을 먼저 보여준다”는 문제 정의는 좋았다. 그러나 지원 직무에 맞추려면 여행자 기록 기능보다 상품 공급 분석과 파트너 발굴 가능성이 더 중요하다. 따라서 이번 차터에서는 스탬프, 개인 사진, 여행 스토리, 상세 일정 생성 같은 여행자용 확장 기능을 제외하고, 데이터 대조와 공급 갭 분석에 집중한다.

## 3. 해결 개념

```text
도시 선택
→ MCP 상품 데이터 수집
→ 공식 관광 자산 수집
→ 명칭 정규화·경험 앵커 생성
→ MCP 상품 경험과 공식 자산 매칭
→ 일치도·공급 갭 점수 계산
→ 대표 경험·상품화 부족 자산·플랫폼 강점 영역 분류
→ 신규 상품/파트너/프로모션 후보 리포트 생성
```

분석의 단위는 상품이 아니라 `경험 앵커`다. 예를 들어 `유니버설 스튜디오 재팬 입장권`, `USJ 익스프레스 패스`, `오사카 테마파크 투어`는 판매 방식은 다르지만 `유니버설 스튜디오 재팬`이라는 하나의 경험 앵커로 묶을 수 있다. 반대로 `도톤보리 야간 산책 투어`는 투어 상품명 그대로가 아니라 `도톤보리`라는 지역 자산과 매칭한다.

## 4. 핵심 사용자

### 4.1 1차 사용자

- 마이리얼트립 T&A 해외 사업개발 매니저
- 지역별 상품 기획·운영 담당자
- 신규 파트너 소싱 담당자
- 프로모션·콘텐츠 기획 담당자

### 4.2 사용 상황

- 특정 도시의 상품 공급 현황을 빠르게 파악할 때
- 공식 관광 자산 대비 마이리얼트립 상품이 부족한 영역을 찾을 때
- 소도시 또는 신규 지역 진입 가능성을 검토할 때
- 기존 인기 상품 외에 새로운 콘텐츠 후보를 찾을 때
- 지역별 프로모션 메시지와 파트너 제안 근거를 만들 때

## 5. MVP 범위

### 5.1 도시 범위

1차 MVP는 대도시와 소도시를 함께 포함한다. 오사카·후쿠오카처럼 상품 공급이 상대적으로 많은 도시만 보면 “없는 지역에서 어떻게 판단할 것인가”를 검증하기 어렵기 때문이다.

#### 1차 MVP 후보

| 구분 | 도시/지역 | 선정 이유 |
|---|---|---|
| 공급 풍부 도시 | 오사카 | 마이리얼트립 T&A 상품이 많아 대표 경험과 중복 상품 구조를 보기 좋다. |
| 공급 중간 도시 | 후쿠오카 | 야타이·도심 미식·근교 자산이 강하고 공식 데이터와 상품 매칭 검증에 적합하다. |
| 소도시/근교 후보 | 나라 | 공식 관광 자산은 강하지만 오사카 대비 상품 공급 차이를 보기 좋다. |
| 소도시/온천 후보 | 벳푸·유후인 | 지역 매력은 분명하지만 상품화 방식이 투어·교통·온천으로 나뉘는지 확인하기 좋다. |
| 소도시/역사 후보 | 히로시마 | 평화기념공원·미야지마 등 공식 자산과 상품 공급 매칭을 보기 좋다. |

### 5.2 확장 후보

- 삿포로·오타루
- 교토
- 가나자와
- 나가사키
- 구마모토
- 부산
- 전주
- 여수

국내 도시는 포트폴리오 보강용 비교군으로 두되, 지원 직무가 T&A 해외 BD인 점을 고려해 1차 MVP의 중심은 일본 도시·소도시로 둔다.

## 6. 데이터 소스

### 6.1 MCP 상품 데이터

마이리얼트립 MCP에서 도시별 T&A 상품을 조회한다.

수집 필드:

- 상품 ID
- 상품명
- 도시
- 카테고리
- 가격 또는 가격대
- 후기 수/평점이 제공되는 경우 해당 값
- 상품 URL
- 상품 설명·포함사항·일정 등 상세 근거

사용 원칙:

- 상품명과 상세에 직접 등장한 표현만 경험 앵커 후보로 사용한다.
- 입장권·패스·투어·예약·특가 같은 판매 문구는 경험 앵커에서 제거한다.
- 불포함사항은 긍정 근거로 사용하지 않는다.
- 단순 이동, 유심, 보험, 여행용품은 T&A 경험 공급 분석에서 제외한다.

### 6.2 공식 관광 자산 데이터

도시·지자체·관광청·공공 데이터에서 공식 관광 자산을 수집한다.

예상 소스:

- 일본 지자체/관광청 공식 관광 페이지
- 공식 관광 데이터 CSV/API
- BODIK/CKAN 등 공개 데이터셋
- 후쿠오카 야타이 공개 데이터
- 오사카 관광 Map Navi 계열 데이터
- 도쿄 관광 공식 데이터

수집 필드:

- 공식 자산 ID
- 명칭
- 도시/지역
- 유형: 명소, 음식, 축제, 체험, 쇼핑, 야간 콘텐츠, 자연, 역사, 교통형 관광
- 공식 설명
- 위치 정보가 있으면 위도/경도
- 공식 URL
- 제공기관
- 갱신일

사용 원칙:

- 공식 출처가 확인되는 데이터만 사용한다.
- 공식 자산이 있다고 해서 마이리얼트립 예약 가능 상품으로 표현하지 않는다.
- 출처와 갱신일이 불명확하면 검수 필요로 둔다.

## 7. 분석 단위와 분류

### 7.1 경험 앵커

경험 앵커는 상품명이나 관광지 설명에서 추출한 실제 방문·체험 대상이다.

예시:

- 유니버설 스튜디오 재팬
- 슈퍼 닌텐도 월드
- 아베노 하루카스 300
- 도톤보리
- 구로몬시장
- 후쿠오카 야타이
- 오호리공원
- 나라공원
- 벳푸 지옥온천
- 미야지마

### 7.2 제외 대상

- `입장권`, `패스`, `투어`, `티켓`, `특가`, `예약` 같은 판매 형식
- `포함 여부`, `불포함`, `주의사항` 같은 확인 문구
- 공항 픽업, 유심, 보험, 여행용품
- 숙소
- 공식 근거와 상품 근거가 모두 부족한 일반 추천

## 8. 핵심 지표

| 지표 | 의미 | 활용 |
|---|---|---|
| 공식 자산 수 | 도시가 공식적으로 보유·홍보하는 관광 자산 수 | 도시 매력의 외부 기준 |
| MCP 연결 상품 수 | 해당 경험 앵커와 연결되는 마이리얼트립 상품 수 | 플랫폼 공급 강도 |
| 공식-MCP 일치율 | 공식 자산 중 MCP 상품과 연결된 비율 | 상품화 커버리지 |
| 공급 갭 점수 | 공식 자산 강도 대비 MCP 상품 공급이 부족한 정도 | 신규 상품/파트너 후보 |
| 반복 근거 수 | 여러 상품에서 같은 앵커가 반복 등장한 횟수 | 대표 경험 판단 |
| 카테고리 다양도 | 투어·입장권·미식·액티비티 등 구매 방식 다양성 | 프로모션/상품 구성 판단 |

## 9. 결과 분류

| 분류 | 조건 | BD 해석 |
|---|---|---|
| 검증된 대표 경험 | 공식 자산에도 있고 MCP 연결 상품도 많음 | 안정적 프로모션·대표 콘텐츠 후보 |
| 상품화 부족 자산 | 공식 자산은 강하지만 MCP 연결 상품이 적음 | 신규 파트너 소싱·상품 기획 후보 |
| 플랫폼 강점 영역 | MCP 상품은 많지만 공식 자산 근거는 약함 | 플랫폼 고유 공급/파트너 네트워크 강점 |
| 과잉 중복 영역 | 같은 경험에 유사 상품이 과도하게 많음 | 상품 정리·차별화·가격 비교 개선 후보 |
| 검수 필요 | 명칭이 애매하거나 도시/경험 연결이 불확실함 | 운영자 검수 큐 |
| 제외 | 판매 문구, 단순 이동, 숙소, 유심, 보험 등 | 분석 대상 제외 |

- 최소 5개 일본 도시/지역을 비교한다.
- 각 도시에서 MCP 상품 데이터와 공식 관광 자산 데이터를 분리 저장한다.
- 공식 자산과 MCP 상품을 같은 경험 앵커로 매칭한다.
- 도시별 공식-MCP 일치율을 계산한다.
- 도시별 상품화 부족 자산 TOP 후보를 보여준다.
- 도시별 검증된 대표 경험 TOP 후보를 보여준다.
- 매칭 근거와 제외 사유를 함께 저장한다.
- 근거 없는 일반 추천을 상품화 후보로 표현하지 않는다.
- 분석 결과를 신규 파트너/상품/프로모션 후보 문장으로 요약한다.

## 14. 안전 및 품질 원칙

- 공식 출처가 없는 관광 자산은 공식 자산으로 표시하지 않는다.
- MCP 상품 근거가 없는 항목은 예약 가능 상품으로 표시하지 않는다.
- 상품 수를 실제 방문객 수나 한국인 방문 비중으로 해석하지 않는다.
- 일치율은 공식 자산과 MCP 상품 간의 데이터 매칭 비율이지 시장 점유율이 아니다.
- 가격·재고·예약 가능 여부는 최종 상품 페이지에서 확인해야 한다.
- 자동 매칭 결과는 신뢰도와 검수 상태를 함께 표시한다.
- 불포함사항, 주의사항, 주변 언급만으로 긍정 매칭하지 않는다.
- 분석 결과는 사업개발 가설이며 실제 매출 개선을 단정하지 않는다.

## 16. 포트폴리오 메시지

> 기존 Codex 플러그인은 여행자에게 도시별 경험을 먼저 보여주는 도구였습니다. 그러나 T&A 해외 사업개발 직무 관점에서 다시 보니 핵심은 여행 기록 기능이 아니라, 마이리얼트립 상품 공급과 공식 관광 자산 사이의 차이를 발견하는 것이었습니다. 그래서 프로젝트를 `T&A Supply Gap Map`으로 재설계했습니다. 이 도구는 MCP 상품 데이터와 지자체·관광청 공식 데이터를 경험 앵커 단위로 매칭해, 도시별 대표 경험과 상품화가 부족한 로컬 자산을 구분합니다. 이를 통해 신규 파트너 소싱, 상품 기획, 프로모션 소재 발굴의 우선순위를 데이터로 설명할 수 있습니다.
