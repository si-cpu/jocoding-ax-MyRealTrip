# Official Tourism Data Sources for T&A Supply Gap Map

- 작성일: 2026-07-18
- 목적: 마이리얼트립 MCP 상품 데이터와 대조할 공식 관광 자산 데이터 후보를 정리한다.
- 사용 원칙: MVP에서는 바로 수집·저장 가능한 공개 CSV/API/공식 페이지를 우선하고, 시장화 목적이 아니므로 과도한 제휴형·상용 데이터는 제외한다.

공식 출처와 파일/API 접근성이 비교적 명확하고, MCP 상품과 경험 앵커 매칭에 바로 사용할 수 있는 데이터다.

| A2 | 일본 지자체 표준 오픈데이터셋 | 일본 지자체 전반 | 관광시설, 이벤트, 문화재 | 공식 관광 자산 표준 구조 | 도시별 포털을 찾는 기준 |

## 3. 수집 대상 데이터셋

MVP에서는 다음 세 종류를 우선 수집한다.

1. 관광시설 목록
   - 명소, 시설, 공원, 전망대, 박물관, 문화시설, 자연 자산
2. 이벤트 목록
   - 축제, 기간성 행사, 야간 행사, 지역 시즌 콘텐츠
3. 문화재 목록
   - 사찰, 신사, 성, 역사 유적, 지정 문화재

후순위로 수집한다.

1. 공식 먹거리/인증 식당
   - 후쿠오카 야타이처럼 도시 정체성과 직접 연결되는 경우만 우선
2. 관광 통계
   - 공급 갭 점수 자체가 아니라 도시 수요 보조 설명에 사용
3. 교통·숙박·상업 시설
   - 이번 MVP의 경험 앵커 분석에서는 제외하거나 보조 필드로만 사용

## 5. MVP에서 피해야 할 해석

- 공식 자산 수가 많다고 여행 수요가 크다고 단정하지 않는다.
- MCP 상품 수가 많다고 실제 방문객 수가 많다고 단정하지 않는다.
- 공식 자산과 MCP 상품이 매칭되지 않았다고 반드시 사업성이 없다고 판단하지 않는다.
- 일반 블로그 추천이나 비공식 랭킹을 공식 자산으로 사용하지 않는다.
- 음식점 영업허가 데이터는 관광 매력 데이터가 아니므로 인증 식당처럼 표현하지 않는다.

## 6. 다음 실행 순서

1. 오사카·후쿠오카·히로시마는 공개 데이터 파일/API URL을 먼저 확정한다.
2. 나라·벳푸·유후인은 공식 관광 자산을 오픈데이터로 받을 수 있는지 확인한다.
3. 데이터가 부족한 소도시는 공식 관광 사이트의 명소 목록을 수동 seed로 시작하고, 출처를 명확히 남긴다.
4. MCP 상품 데이터와 공식 자산 데이터를 같은 `experience_anchor` 구조로 정규화한다.
5. 도시별 일치율과 공급 갭 점수를 계산한다.
