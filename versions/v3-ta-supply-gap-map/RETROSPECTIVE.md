# T&A Supply Gap Map 프로젝트 회고

- 회고 기준일: 2026-07-19
- 프로젝트 맥락: 마이리얼트립 T&A 해외 사업개발 직무 포트폴리오
- 현재 단계: 후쿠오카·히로시마 1차 분석과 재현 파이프라인 구축 완료
- 한 줄 회고: 여행자용 경험 탐색 플러그인에서 출발했지만, 범위를 넓히는 실수를 인정하고 공식 관광자원과 마이리얼트립 상품 공급을 장소 계층 단위로 대조하는 재현 가능한 사업개발 분석 도구로 다시 좁혔다.

## 1. 무엇을 만들었나

이 프로젝트는 지자체·공공기관이 제공하는 공식 관광 장소와 마이리얼트립 T&A 상품을 대조한다. 상품의 판매 형식이 아니라 실제 방문 장소를 기준으로 다음을 구분한다.

- 여러 상품에서 반복적으로 확인되는 대표 경험
- 공식 관광자원이면서 마이리얼트립 상품과 연결되는 장소
- 공식 관광자원이지만 현재 수집 범위에서는 연결이 확인되지 않은 장소
- 신규 투어·입장권·파트너 소싱 검토가 필요한 후보
- 명칭 불일치, 상세정보 부족, 요청 제한 때문에 추가 검증이 필요한 항목

핵심 산식은 단순하다.

```text
공식 관광 장소
− 마이리얼트립에서 확인된 장소별 상품 공급
= 직접 재검색 및 신규 상품개발 검토 후보
```

그러나 실제 구현에서는 상품 제목만 비교하면 안 됐다. 입장권은 제목에 장소가 드러나지만, 투어의 방문지는 상세 일정에 숨어 있기 때문이다. 최종적으로 상품 검색 결과, MCP 상세정보, 공개 상품 페이지의 구조화 일정까지 결합해 장소별 공급 근거를 만들었다.

또한 이 산식은 한 방향으로만 보면 불완전했다. 공식 관광지에는 없지만 MCP에는 존재하는 리버 크루즈·야간 체험 같은 민간 콘텐츠도 있기 때문이다. 최종 분석은 다음 두 질문을 함께 다룬다.

- 공식 관광자원 중 마이리얼트립 상품으로 연결되지 않은 곳은 어디인가?
- 마이리얼트립에는 있지만 공식 `관광지` 목록이 포착하지 못한 민간·상업적 경험은 무엇인가?

## 2. 처음의 문제와 스코프 실패

출발점은 `여행 경험 진열대` Codex 플러그인이었다. 사용자가 도시를 입력하면 상품 대신 `뭐 하지`, `뭐 먹지`에 해당하는 경험을 먼저 보여주고 공식 상품으로 연결하는 방식이었다.

이 아이디어를 웹으로 확장하면서 다음 기능이 붙었다.

- 도시 선택과 경험 탐색
- 일정과 지도
- 숙소와 이동
- 사진 스탬프
- 여행 종료 후 스토리 생성
- 내부 행동 데이터와 사업 리포트

각 기능은 개별적으로 의미가 있었지만, 결과적으로 여행자용 제품과 사업개발 분석 도구가 한 프로젝트 안에 섞였다. 큰 제품을 만들수록 지원 직무와의 연결이 선명해질 것이라고 생각했지만 실제로는 반대였다. 기능은 많아졌고, “이 프로젝트가 정확히 어떤 의사결정을 돕는가”라는 질문에는 답하기 어려워졌다.

이번 회고에서 가장 큰 학습은 다음과 같다.

> 좋은 아이디어를 더 많이 붙이는 것이 고도화는 아니다. 핵심 사용자의 한 가지 의사결정을 더 정확하게 만드는 것이 고도화다.

따라서 기존 여행자용 웹 MVP는 보존하고, 웹 확장 전 시점에서 별도 브랜치를 만들어 T&A 공급 공백 분석에만 집중했다.

## 3. 방향을 어떻게 다시 잡았나

마이리얼트립 T&A 해외 사업개발 매니저의 실제 업무와 연결되는 질문으로 범위를 다시 정의했다.

> 지자체가 관광자원으로 관리하는 장소 중 마이리얼트립이 이미 충분한 상품을 가진 곳은 어디이며, 공식 매력에 비해 상품 연결이 약한 곳은 어디인가?

이 질문은 다음 업무로 직접 이어진다.

- 신규 지역·카테고리 발굴
- 현지 파트너 소싱
- 기존 투어 코스 보강
- 신규 테마 투어 기획
- 대표 콘텐츠와 롱테일 콘텐츠 분리
- 도시별 상품 포트폴리오 진단

분석 단위도 상품에서 장소로 바꿨다.

- `도톤보리 야간 산책 투어` → `도톤보리`
- `야타이 포장마차 투어` → `야타이`
- `히로시마·미야지마 일일 투어` → 상세 일정에 실제로 등장하는 개별 장소
- `오리즈루 타워 입장권` → `오리즈루 타워`

이 전환 덕분에 판매 방식이 다른 상품을 하나의 관광 장소 아래에서 비교할 수 있게 됐다.

## 4. 데이터 기준을 세우며 고친 문제

### 4.1 공식 데이터가 많다고 모두 관광지는 아니었다

교토 관광시설 데이터 560건을 처음 확인했을 때 수치 자체를 성과처럼 받아들일 위험이 있었다. 실제로는 식당, 상점, 주차장, 시설 내부 구성요소, 일반 편의시설이 섞여 있었다. 관광지 분석과 무관한 데이터를 제거하지 않으면 공급 공백 후보가 오염된다.

이에 따라 다음 기준을 적용했다.

- 독립적으로 방문할 이유가 있는 장소만 1차 관광자산으로 채택
- `FoodPlace`와 개별 식당은 관광지 분석에서 제외
- 주차장, 연구기관, 고객 전용 정원, 지점형 부속시설 제외
- 동일 장소의 내부 구성요소는 독립 관광지로 중복 계산하지 않음
- 공식 명칭과 한국어 검수명을 함께 보존

### 4.2 별칭을 늘리는 것보다 번역 품질이 중요했다

초기에는 일본어 공식명과 한국어 상품명을 연결하기 위해 수동 별칭을 많이 추가했다. 그러나 별칭이 늘수록 다른 장소와 잘못 연결될 가능성도 커졌다. 이후 공식 일본어명을 검수된 한국어명으로 번역하고, 정말 필요한 동의어만 제한적으로 추가하는 방식으로 바꿨다.

### 4.3 투어 제목만 보면 공급을 과소평가했다

입장권은 제목에 대상 시설이 드러나지만 투어 제목은 `후쿠오카 일일투어`, `히로시마·미야지마 투어`처럼 구성된다. 검색 결과 제목만 비교했을 때 공식 관광지와의 매칭률이 실제보다 낮게 나타났다.

이를 해결하기 위해 후쿠오카·히로시마 투어를 모두 상세 조회했다.

- 검색 결과의 투어 행: 151개
- 중복 제거 후 고유 투어: 145개
- MCP 상세 조회 성공: 145개
- MCP 상세 섹션: 1,231개
- 공개 상품 페이지 조회 성공: 145개
- 구조화 일정 보유 상품: 133개
- 일정: 149개
- 방문 슬롯: 1,004개

MCP 상세는 일부 상품 소개를 `...`로 잘라 반환했다. 공개 상품 페이지의 `__NEXT_DATA__`에는 전체 일정 슬롯이 구조화되어 있었기 때문에 이를 보완 근거로 사용했다.

### 4.4 상세정보도 그대로 믿으면 과잉 연결됐다

상세 조회를 추가한 뒤에도 세 가지 오연결이 발생했다.

1. `원폭 자료관`이라는 일반 표현 때문에 나가사키 상품이 히로시마 평화기념자료관에 연결됐다.
2. 오리즈루 타워가 투어 집결지로만 등장했는데 방문 장소로 계산될 뻔했다.
3. 히로시마성이 불포함 입장료에 등장해 긍정 방문 근거처럼 보일 수 있었다.

다음 규칙으로 수정했다.

- `원폭 자료관`처럼 여러 도시에 존재할 수 있는 명칭은 도시 문맥을 함께 확인
- 집결·미팅·픽업·출발·해산 장소는 방문 근거에서 제외
- 불포함·제외·취소 영역은 긍정 근거로 사용하지 않음
- 투어는 일정 슬롯, 상품 소개, 포함사항에서 방문 근거가 확인될 때만 확정
- 입장권은 상품 제목에 대상 시설이 직접 명시될 때 확정
- 추천 상품 영역은 현재 상품의 일정에서 제외

### 4.5 장소를 1대1로 비교해도 부족했다

MCP 상품에 있는 경험이 공식 관광지 목록에 없다고 판단했지만, 상위 장소와 다른 공식 카탈로그까지 확인하자 결과가 달라졌다.

- 팀랩 포레스트와 산리오 드리밍파크는 `BOSS E·ZO FUKUOKA` 내부 시설이었다.
- 나카스 야카타부네·나카가와 리버 크루즈는 공식 `spots`에는 없었지만 공식 `tours·experience` 카탈로그에 존재했다.
- `후쿠오카 천연온천` 상품은 상세 주소가 기타큐슈시여서 후쿠오카시 공급에서 제외해야 했다.
- `CLUB Rebellion`은 나카스라는 공식 관광 권역 안에 있지만, 공식 관광지와 직접 동일시할 수 없는 사적 야간 콘텐츠였다.

따라서 비교 단위를 다음 계층으로 바꿨다.

```text
도시
└─ 권역·거리
   └─ 테마파크·복합시설
      └─ 개별 장소·시설
         └─ 예약 가능한 체험
            └─ 판매 상품·옵션
```

매칭 결과도 `직접 일치`, `상위 장소에 포함`, `다른 공식 카탈로그`, `공식 권역 안의 민간 확장`, `도시 밖`, `미확정`으로 나눴다. 이 과정을 거치지 않은 `MCP만 있음`은 플랫폼 독자 콘텐츠로 확정하지 않는다.

### 4.6 수작업 분석을 재현 가능한 파이프라인으로 바꿨다

분석을 반복하면서 실행 순서와 결과가 사람의 기억에 의존하는 문제가 생겼다. 이를 해결하기 위해 다음 단계를 하나의 진입점으로 묶었다.

```text
원천 수집
→ 도시·비관광 데이터 필터
→ 공식 장소 앵커 생성
→ 제한된 별칭 생성
→ 투어 상세·공개 일정 매칭
→ 공식 장소와 MCP 상품 연결
→ 장소 계층 검수 게이트
→ 일치율·정확한 리스트·발굴 후보 생성
→ PDF·테스트
```

`rebuild`는 저장된 원천만 사용해 결과를 재현하고, `refresh`만 네트워크와 MCP를 호출한다. 덕분에 요청 제한이나 외부 데이터 변경 없이 매칭 규칙을 안전하게 검증할 수 있다. 장소 계층은 무리하게 자동 확정하지 않고 검수 CSV를 통과해야 보고서가 생성되도록 했다.

## 5. 현재 확인된 결과

### 5.1 분석 범위

| 항목 | 후쿠오카 | 히로시마 |
|---|---:|---:|
| MCP 검색 반환 감사 풀 | 180개 | 20개 |
| 도시별 공식 관광지 분모 | 81곳 | 28곳 |
| 긍정 근거가 확인된 장소 | 10곳 | 4곳 |
| 잠정 확정 일치율 | 12.3% | 14.3% |
| 수집 상태 | MCP 일부 오류·잠정치 | MCP 일부 오류·잠정치 |

후쿠오카는 요카나비 일본어 관광지 468건을 전체 보존하고, 공식 한국어 가이드에도 발행된 후쿠오카시 내 방문 관광지에서 통합 전 중복까지 제거한 81곳을 분석 분모로 재구축했다. 따라서 기존 미술관·박물관 5곳 표본은 더 이상 도시 일치율에 사용하지 않는다.

### 5.2 도시별 확인된 상품 연결

| 공식 장소 | 연결 상품 수 | 근거 |
|---|---:|---|
| 슛케이엔 | 투어 2개 | 전체 일정 슬롯 및 정원 입장료 포함 |
| 히로시마성 | 투어 2개 | 일정에 도착·견학 또는 독립 방문 슬롯 명시 |
| 히로시마 평화기념자료관 | 투어 3개 | 전체 일정 슬롯 및 박물관 입장료 포함 |
| 오리즈루 타워 | 입장권 2개 | 입장권 제목에 대상 시설 직접 명시 |

현재 관측 기준으로 히로시마 공식 장소 28곳 중 4곳이 상품과 연결되어 `잠정 확정 일치율 14.3%`가 나왔다. 다만 장소명 직접 검색과 일부 액티비티 재수집이 남아 있으므로 최종 일치율로 단정하지 않는다.

후쿠오카 공식 관광지 81곳 중 10곳은 입장권 제목 또는 투어 상세의 실제 방문 근거로 연결됐다. 처음에는 9곳으로 집계했지만, 노코노시마 코스가 이동 시간표로 표현되어 `방문`이라는 단어가 없다는 이유로 누락된 것을 발견했다. 코스 제목과 시간대별 입도·체류·출도 문맥을 방문 근거로 인정하되, 단순 집결지는 제외하도록 규칙과 테스트를 수정했다. 현재 12.3%는 MCP 일부 수집 오류가 남은 잠정 하한값이다.

### 5.3 후쿠오카 신뢰도 전수 감사

수치가 예상보다 낮았고 검색 상품의 도시 혼입도 확인되어, 결과를 확대 해석하기 전에 후쿠오카만 별도로 다시 감사했다.

- 검색 반환 180개 모두 MCP 상세 원본을 확보했고 178개에서 상세 섹션을 파싱했다.
- 비투어 42개는 이용 장소·주소를 직접 검토했다. 13개는 후쿠오카시, 29개는 시외 또는 광역 상품이었다.
- 투어를 포함해 보수적으로 후쿠오카시 콘텐츠를 확인한 상품은 25개였고, 126개 투어는 출발지와 방문지를 혼동하지 않기 위해 도시 범위 판정을 보류했다.
- 기존 장소×상품 연결 27건을 재검토해 단독 입장권·고정 일정 21건, 선택 옵션 4건, 단순 언급 1건, 상위 장소 포함 1건으로 분해했다.
- 옵션·언급 연결을 내려도 같은 장소에 다른 엄격 상품이 남아 장소 분자는 10곳, 엄격 일치율은 12.3%로 유지됐다.
- 공식명 직접 일치인데 자동 매칭이 제외한 8쌍을 전수 검토했다. 하차 지점 1건, 대체 일정 2건, 옵션 1건, 지리적 연결 3건, 조망 1건으로 확인되어 엄격 연결에는 추가하지 않았다.
- 상품 안의 `캐널시티 히카타` 오기는 검토 별칭으로 추가해 실제 일정 2건을 복구했다.
- 요카나비의 현재 공식 투어 11건은 장소 81곳과 분리해 전수 저장했다.

따라서 12.3%는 연결된 10곳의 근거 면에서는 이전보다 신뢰할 수 있지만, 오류가 남은 MCP 카테고리 9개와 미등록 번역 별칭 때문에 최종 시장 일치율은 아니다. 상세 내용은 `data/supply_gap_analysis/reports/FUKUOKA_TRUST_AUDIT.md`에 남겼다.

## 6. 이 결과가 사업개발에 주는 의미

이 도구의 결과는 곧바로 “이 투어를 출시해야 한다”는 정답이 아니다. 대신 사업개발자가 검증할 후보와 근거를 빠르게 만든다.

### 6.1 기존 투어 보강

현재 인기 투어의 동선 주변에 공식 관광자원이 있지만 상품 일정에서 거의 반복되지 않는다면, 기존 파트너에게 코스 추가 또는 시즌형 변형을 제안할 수 있다.

### 6.2 신규 테마 투어

현재 미연결 후보를 주제로 묶어 상품 가설을 만들 수 있다.

- 과학·가족 코스: 에바야마 기상관 → 누마지 교통박물관 → 어린이문화과학관
- 로컬 역사 코스: 라이 산요 사적자료관 → 향토자료관 → 유화원
- 현장형 평화 코스: 혼카와초등학교 평화자료관 → 후쿠로마치초등학교 평화자료관 → 평화기념자료관
- 서브컬처 코스: 히로시마시 만화도서관 → 현대미술관 → 인근 상권

이 코스들은 아직 상품 제안이 아니라 `검증할 사업 가설`이다. 실제 개발 전에는 이동시간, 영업일, 단체 수용, 한국어 운영, 파트너 의향, 예상 가격과 수요를 확인해야 한다.

### 6.3 파트너 소싱 우선순위

공식성은 높지만 연결 상품이 없는 장소는 다음 파트너 후보를 찾는 출발점이 된다.

- 시설 직접 계약
- 현지 가이드·랜드사
- 교통 사업자
- 지역 DMO·관광협회
- 테마 전문 운영자

### 6.4 공급 포트폴리오 진단

한 장소에 여러 상품이 반복되면 대표 경험으로 볼 수 있다. 반대로 공식 장소는 많지만 상품이 특정 장소에만 집중되면 도시의 매력을 충분히 상품화하지 못하고 있다는 가설을 세울 수 있다.

## 7. 잘한 점

### 문제를 다시 좁혔다

이미 많은 기능을 만든 상태에서 계속 확장하지 않고, 기존 웹을 보존한 채 별도 브랜치에서 분석 도구로 재정의했다. 기능을 버린 것이 아니라 제품의 목적을 분리했다.

### 결론보다 근거를 저장했다

모든 연결에 상품 ID, 공식명, 한국어명, 상품 URL, 상세 섹션, 일정 슬롯과 근거 문장을 남겼다. 결과가 틀렸을 때 원인을 추적할 수 있다.

### 실패한 요청을 데이터 부재로 해석하지 않았다

MCP 요청 제한, 상세정보 잘림, 도시 혼입을 별도 상태로 남겼다. 수집 실패와 상품 부재를 구분한 것은 공급 갭을 과장하지 않기 위한 핵심 원칙이었다.

### 실제 반례로 규칙을 강화했다

나가사키 원폭자료관 오매칭, 오리즈루 타워 집결지, 히로시마성 불포함 문구, 노코노시마 일정 누락을 발견하고 도시 문맥·집결지 제외·부정 영역 차단·시간표형 코스 인식 규칙을 추가했다.

### 분석을 다시 실행할 수 있게 만들었다

저장 데이터 재계산, 네트워크 갱신, 보고서 생성, 테스트를 모드별로 분리했다. 각 단계의 성공·실패·소요시간과 도시별 지표를 실행 요약 JSON으로 남겨 결과가 어떤 순서로 만들어졌는지 추적할 수 있다.

## 8. 부족했던 점

### 처음부터 분석 단위를 고정하지 못했다

상품, 경험, 장소, 투어 코스가 혼재해 수동 앵커가 늘었고 데이터가 오염됐다. `무조건 장소 단위`라는 원칙을 더 일찍 고정했어야 했다.

### 도시 범위를 먼저 넓혔다

오사카·교토·도쿄·서울·부산·여수·대전까지 확장하려 했지만, 한 도시의 수집과 매칭 품질도 완성되지 않은 상태였다. 후쿠오카·히로시마 두 도시에 집중한 뒤에야 상세 조회와 오연결 문제를 발견할 수 있었다.

### 공식 데이터의 대표성이 도시마다 달랐다

히로시마와 후쿠오카 모두 도시별 분모를 갖췄지만 원천의 수록 정책이 다르다. 후쿠오카는 공식 한국어 발행 여부까지 적용한 81곳이고, 히로시마는 공식 관광시설 공개데이터를 정제한 분모이므로 동일한 `연결률`을 단순 도시 인기 순위로 해석하지 않는다.

### 공식 관광지 목록도 전체 관광 경험이 아니었다

공식 데이터는 공원·신사·박물관·문화재 같은 공공 관광자원에는 강하지만, 크루즈·야간 콘텐츠·민간 체험·복합시설의 하위 콘텐츠를 놓칠 수 있다. 반대로 MCP 검색은 출발 도시를 기준으로 기타큐슈·유후인·벳푸 상품까지 섞을 수 있다. 한 데이터가 다른 데이터보다 우월하다고 가정하지 않고, 서로 다른 수록 목적을 가진 두 카탈로그의 교집합과 차집합으로 해석해야 한다.

### 수요 검증은 아직 없다

공식 관광자원과 상품 공급의 차이는 사업 기회를 보여줄 수 있지만, 고객 수요를 증명하지 않는다.

수요 검증을 이번 1차 결과에 포함하지 않은 것은 데이터가 없어서가 아니다. 외부에서 확인할 수 있는 리뷰 수, 평점, 검색 노출과 같은 값은 실제 예약 의향뿐 아니라 상품 판매 기간, 노출 순위, 프로모션, 후기 작성률의 영향을 함께 받는 간접 지표다. 이 값을 정제 없이 수요로 해석하면 오염도가 높은 추정치가 되고, 상품개발과 파트너 소싱의 우선순위를 오히려 잘못 정할 위험이 있다고 판단했다.

따라서 이번 단계에서는 공식 관광자원과 실제 상품 공급 사이의 차이를 정확하게 찾는 데 집중하고, 수요는 검증되지 않은 상태로 명시했다. 실제 예약 수, 상세페이지 조회, 클릭률, 예약 전환율, 취소율과 같은 확정적 데이터에 접근할 수 있는 마이리얼트립 내부에서 다음 가설을 검증하고 싶다.

> 공식 관광자원으로서 매력은 확인되지만 현재 상품 연결이 적은 장소 중, 실제 고객 행동 데이터에서도 신규 상품개발 가능성이 나타나는 곳은 어디인가?

이후에는 내부 수요 데이터에 가격, 계절성, 이동 편의성, 파트너 운영 가능성을 함께 결합해 최종 상품개발 우선순위를 산정해야 한다.

## 10. 최종 회고

이 프로젝트의 가장 큰 성과는 새로운 기능을 많이 만든 것이 아니다. 잘못된 스코프를 인정하고, 사업개발자가 실제로 사용할 수 있는 질문으로 프로젝트를 다시 설계한 것이다.

처음에는 여행자에게 더 많은 경험을 보여주는 제품을 만들고 싶었다. 지금은 한 단계 앞에서, 플랫폼이 어떤 도시 매력을 충분히 상품화했고 무엇을 놓치고 있는지를 근거로 보여주는 도구가 됐다. 이 결과는 여행자에게 새로운 선택지를 제공하는 일과 상품개발자가 새로운 콘텐츠를 발굴하는 일을 연결한다.

아직 “이 장소는 반드시 팔린다”고 말할 수는 없다. 대신 다음과 같이 말할 수 있다.

> 공식 관광자원, 현재 상품 공급, 실제 투어 일정이라는 세 가지 근거를 대조해 상품개발 검토가 필요한 장소를 좁혔고, 오연결과 데이터 부족을 분리해 다음 검증 순서를 만들었다.

이제 여기에 한 문장을 더할 수 있다.

> 같은 입력 데이터와 검수 원장으로 분석을 다시 실행해 같은 분류와 보고서를 만들 수 있도록 파이프라인화했다.

이 문장이 현재 프로젝트가 증명한 범위다. 다음 단계는 후보를 더 많이 만드는 것이 아니라 시카노시마 또는 아부라야마 한 곳을 실제 파트너·동선·가격 가설까지 연결해 검증하는 것이다.
