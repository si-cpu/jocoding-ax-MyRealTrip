# 여행 콘텐츠 지도 — Codex 플러그인 v0.5

목적지를 입력하면 마이리얼트립의 투어·티켓·체험·식사 상품을 분석해 `뭐 하지?`와 `뭐 먹지?`를 구체적인 콘텐츠 키워드로 펼쳐주는 플러그인입니다.

## 해결하려는 문제

목적지를 정한 여행자는 그곳에서 무엇을 하고 먹을지 구체적으로 떠올리지 못해 긴 상품 목록을 반복해서 살펴보게 됩니다. 기존의 `투어·입장권·액티비티` 같은 상품 유형 대신 `슈퍼 닌텐도 월드·아스널·오코노미야키`처럼 여행 목적이 그려지는 단위로 상품을 다시 색인합니다.

이 플러그인은 사용자를 대신해 최적 상품을 추천하지 않습니다. 가능한 경험을 먼저 펼치고 사용자가 관심 키워드를 직접 선택하게 합니다.

## 사용법

여행지만 입력하면 됩니다. 날짜는 선택사항입니다.

```text
오사카에서 뭐 하고 뭐 먹을 수 있어?
런던 상품을 해리포터·축구·공연 같은 키워드로 보여줘.
2026-08-10 부산에서 예약 가능한 경험을 찾아줘.
```

첫 결과는 다음처럼 키워드 중심으로 표시됩니다.

```text
오사카에서 뭐 하지?
- 유니버설 스튜디오 재팬
- 슈퍼 닌텐도 월드
- 오사카성

오사카에서 뭐 먹지?
- 오코노미야키
- 타코야키
- 모던야키
```

사용자가 키워드를 선택하면 그 키워드가 실제로 명시된 마이리얼트립 상품과 공식 링크를 펼칩니다.

## 작동 절차

1. 여행지 이름으로 마이리얼트립 TNA 상품을 페이지 단위로 조회합니다.
2. 검색 응답의 `도시 ∙ 카테고리`를 이용해 다른 도시 상품을 제외합니다.
3. 이동·교통, 유심, 보험 등 경험 탐색과 무관한 상품을 제외합니다.
4. 상품 상세의 제목, 소개, 포함사항, 일정 제목·설명을 수집합니다.
5. 장소·시설·작품·캐릭터·스포츠팀·공연·음식·구체적인 행동을 후보로 추출합니다.
6. `추천·특가·일일투어·입장권` 같은 판매·상품 형식 표현을 제거합니다.
7. 검증된 동의어만 통합하고 `뭐 하지?`와 `뭐 먹지?`로 나눕니다.
8. 각 키워드에 상품 ID, 원문 근거와 공식 `productUrl`을 연결합니다.
9. 링크를 보여주기 직전에 HTTPS·공식 도메인·HTTP 응답·최종 리디렉션을 확인합니다.
10. 날짜가 입력되면 최종 상품마다 옵션 API를 조회해 해당 날짜의 옵션 존재 여부를 확인합니다.

## 판단 기준

- 상품명·소개·일정에 직접 등장하거나 검증된 공식 별칭일 때만 키워드와 상품을 연결합니다.
- `애니메이션 관련 상품`이라는 이유만으로 특정 작품인 `나루토`를 생성하지 않습니다.
- 하나의 상품에서 여러 키워드를 꺼낼 수 있지만, 각 키워드는 독립된 원문 근거를 가져야 합니다.
- `오코노미야키`와 `모던야키`, `해변열차`와 `스카이캡슐`처럼 다른 경험을 과도하게 합치지 않습니다.
- 음식은 식사권, 식당 예약, 푸드투어, 쿠킹클래스 또는 포함 식사처럼 상품과 직접 연결되는 경우를 우선합니다.
- 전체 페이지나 상세 조회가 제한되면 `이번에 확인한 상품 범위에서`라고 표시합니다.

세부 규칙은 `skills/map-travel-content/references/content-indexing-rules.md`에 있습니다.

## 정보 부족과 실패 처리

- API 키가 없고 공식 MCP도 사용할 수 없으면 일반 웹 정보로 결과를 대체하지 않습니다.
- 상품 상세가 없으면 검색 상품명에서 직접 확인되는 키워드만 사용합니다.
- 선택 날짜의 옵션이 비어 있으면 해당 날짜에 예약 가능하다고 표현하지 않습니다.
- 먹거리 키워드는 유명하더라도 마이리얼트립 상품 근거가 없으면 상품 링크를 만들지 않습니다.
- 상품 URL이 공식 도메인이 아니거나 접근 실패·외부 리디렉션이 발생하면 링크를 제외합니다.
- 예약·결제·취소는 수행하지 않으며 최종 가격과 재고는 상품 페이지에서 확인하게 합니다.

## 데이터 연결

- Partner REST API: `https://partner-ext-api.myrealtrip.com`
- 공식 MCP: `https://mcp-servers.myrealtrip.com/mcp`
- REST 인증: `MYREALTRIP_API_KEY` 환경변수

API 키를 프롬프트, 코드, `.mcp.json`, 테스트 또는 제출 로그에 입력하지 마세요.

## 현재 검증 범위

- 페이지 순회 및 `gid` 중복 제거
- 요청 도시 불일치 상품 제거
- 교통·유심 등 유틸리티 상품 제거
- HTML이 포함된 상세 설명의 텍스트 정리
- 상세 근거와 공식 URL 보존
- 날짜 옵션이 없는 상품을 예약 가능으로 표시하지 않는 로직
- 비공식 URL과 외부 도메인 리디렉션 차단
- 실제 마이리얼트립 상품 URL의 HTTP 200 응답 확인
- 오사카·부산·런던·충칭·전주·삿포로 공개 상품 표본의 콘텐츠 추출 가능성

이번 제출은 개발자 자체 테스트만 수행합니다. 실제 여행자를 대상으로 한 비교 실험은 일정상 후속 검증으로 남기며, 탐색 시간·상품 클릭·결제 전환 개선을 입증된 효과로 주장하지 않습니다.

실제 Partner REST 호출은 로컬 환경에 API 키를 설정한 뒤 추가 검증해야 합니다.

## 제한사항

- 상품 설명의 품질과 범위에 따라 키워드 수가 달라집니다.
- 상품 수가 많은 도시는 응답시간과 호출량 때문에 일부 상세 조회가 제한될 수 있습니다.
- 먹거리 예약 상품이 적은 도시에서는 `뭐 먹지?` 결과가 비어 있을 수 있습니다.
- 키워드 탐색이 결제 전환을 높인다는 효과는 아직 입증하지 않았습니다.
- 개인화, 예약 이력 조회, 항공·숙소, 자동 일정 생성은 MVP 범위가 아닙니다.
