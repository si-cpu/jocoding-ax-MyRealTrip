# 출처 기록 및 검증 규칙

## 원칙

기획자가 같은 원문을 다시 열고 판단을 재검토할 수 있어야 한다. 링크만 나열하지 말고 어떤 사실을 어느 위치에서 확인했는지 연결한다.

## 출처 확인 순서

1. 공식 상품·기능·정책·회사 페이지와 원자료
2. 조사 방법과 표본이 공개된 논문·통계·보고서
3. 원 인터뷰나 발표를 포함한 신뢰할 수 있는 보도
4. 다수의 독립적인 공개 리뷰·사례
5. 검색 결과 요약과 출처 불명 자료

마지막 항목은 원문을 찾기 위한 단서로만 사용한다.

## 출처 원장 필드

각 출처에 `S1`, `S2`처럼 ID를 부여하고 다음을 기록한다.

- 문서의 정확한 제목
- 발행 주체 또는 저자
- 발행일 또는 마지막 수정일. 찾을 수 없으면 `표기 없음`
- 실제로 원문을 확인한 날짜
- 검색 결과가 아닌 원문 URL
- 근거가 있는 절, 표, 페이지, 문단 또는 화면 영역
- 이 출처에서 직접 확인한 사실 ID `F#`
- 조사 대상·지역·시점·표본·이해관계 등 적용 한계

## 사용 금지

- 검색 결과 요약문만 보고 `FACT`로 기록하지 않는다.
- URL이 없는 외부 주장을 사실로 기록하지 않는다.
- 다른 대상을 조사한 연구 결과를 마이리얼트립 사용자에게 직접 일반화하지 않는다.
- 보도자료의 회사 주장을 독립 검증된 사용자 성과처럼 표현하지 않는다.
- 출처가 말하지 않은 인과관계, 빈도, 시장 규모 또는 매출 효과를 덧붙이지 않는다.

## 접근 실패

원문을 열 수 없으면 다음처럼 기록한다.

```text
SEARCH_LEAD L1
후보 제목:
후보 URL:
확인하지 못한 이유:
확인이 필요한 주장:
```

`SEARCH_LEAD`는 후속 조사 목록이다. 명제 지지·반박이나 `GO/REVISE/HOLD/DROP` 결정에 사용하지 않는다.

## 인용 표현

본문의 사실 뒤에 `[F1/S1]`처럼 사실과 출처 ID를 함께 붙인다. 출처 원장에서는 제목을 클릭 가능한 원문 링크로 제공한다. 짧게 요약하고, 판단에 꼭 필요한 경우에만 짧은 원문 구절 또는 정확한 수치를 기록한다.
